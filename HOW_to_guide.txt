🚀 HOW TO EXECUTE — Astraal Orientation Module
1️⃣ System Overview

This module:

Is non-adaptive

Uses rule-based logic only

Runs via PHP + Python CLI

Is shared-hosting safe

Is stateless after onboarding

Does not connect to adaptive systems

Execution Flow:

Learner → Welcome → Questions → Save Inputs
        → Python rule engine
        → Suggestions stored
        → Learner chooses
        → Clean exit

2️⃣ Environment Requirements
Server Requirements

PHP 7.4+

MySQL 5.7+

Python 3.8+

Shared hosting compatible (GoDaddy, UWAMP, etc.)

Python Dependencies

No external ML libraries required.

Only:

pip install mysql-connector-python

3️⃣ Step-by-Step Deployment
STEP 1 — Import Database

Open:

phpMyAdmin

OR MySQL CLI

Import:

/database/orientation_schema.sql


Verify tables:

learners
orientation_inputs
orientation_suggestions
orientation_log

Edit:

/public_html/config/db.php

/python/orientation_engine.py

database="astraal_lxp"

STEP 5 — Production Execution (Live Mode)

When learner clicks:

Show my starting suggestions


This runs:

exec("python3 /home/username/python/orientation_engine.py $learner_id");


Then redirects to suggestions page.

4️⃣ Cron: Automatic Cleanup (Recommended)

Add in cPanel → Cron Jobs:

php /home/username/public_html/cron/cleanup_expired_orientation.php


Schedule:

Once per month

This removes:

Orientation inputs older than 30 days

Suggestions older than 30 days

Maintains stateless design.

7️⃣ Common Errors & Fixes
❌ “python3 not found”

Fix:

Use full path:

exec("/usr/local/bin/python3 /home/username/python/orientation_engine.py $learner_id");

❌ Suggestions not appearing

Check:

SELECT * FROM orientation_inputs;


If empty → save_inputs not running.

Check:

SELECT * FROM orientation_suggestions;


If empty → Python engine failed.

❌ Permission denied

Run:

chmod +x orientation_engine.py
